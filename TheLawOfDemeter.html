최소지식원칙(Principle of least knowledge)혹은 데메테르의법칙(The law of Demeter)라고 불리는 원칙은 객체 지향 디자인원칙중의 하나다. 결합도가 낮은 설계를 목적으로한다. 왜 결합도가 낮을 필요가 있을까? 만약 한 클래스를 여러 클래스들이 상속하고있고, 그 클래스안에서 여러클래스들의 메소드를 불러오고 있다고 한다면, 그 클래스를 수정하게 되었을 경우 관계되는 클래스들의 수정이 필요로 하게된다.

이원칙에서는 한메소드에서 다음과같은 종류의 객체에있는 메소드들을 실행하라는 가이드라인을 제시한다.

  • 자기자신의 메소드
  • 메소드에 매개변수로 전달된 객체
  • 그 메소드에서 생성하거나 인스턴스를 만든 객체
  • 그 객체에 속하는 구성요소
class Car
{
public:
    //이 클래스의 구성요소. 메소드 호출가능
    Engine engine;
    Car(){};
    void start(Key key)
    {
        // 메소드내의 새로운객체. 메소드 호출가능
        Doors doors = new Doors;

        // 메소드의 매개변수. 메소드 호출가능
        bool authorized = key.turns();
        if(authorized)
        {
            engine.start();
            // 객체의 구성요소. 메소드 호출가능
            updateDashboardDisplay();
            doors.lock();
        }
    }
    void updateDashboardDisplay(){};
}

위의 소스코드는 헤드퍼스트의 예제이다.

단점

객체들간의 의존성과 유지보수가 용이해지나, 메소드호출을 처리하기 위한 래퍼클래스의 필요성과 시스템의 복잡도 증가, 성능저하가 있을 수 있다.
