strategy pattern

행위 소프트웨어 디자인패턴이며, 알고리즘을 캡슐화하여 상호 교체가 가능하게만든다. 이 패턴을 사용하면 클라이언트와는 독립적으로 알고리즘을 변경할 수 있다. 그리고 동적으로 알고리즘을 교체할 수 있는 구조로 되어있다.

구조

실제 예

struct Logger {

    enum LogStyle {
        case lowercase
        case uppercase
        case capitalized
    }

    let style: LogStyle

    func log(_ message: String) {
        switch style {
        case .lowercase:
            print(message.lowercased())
        case .uppercase:
            print(message.uppercased())
        case .capitalized:
            print(message.capitalized)
        }
    }
}

여기서 style을 10개이상으로 추가하려면 logger를 직접건드려 enum과 log함수를 수정해야한다.

// What
protocol LoggerStrategy {
    func log(_ message: String)
}

// Who
struct Logger {
    let strategy: LoggerStrategy

    func log(_ message: String) {
        strategy.log(message)
    }
}

// How
struct LowercaseStrategy: LoggerStrategy {
    func log(_ message: String) {
        print(message.lowercased())
    }
}

struct UppercaseStratgy: LoggerStrategy {
    func log(_ message: String) {
        print(message.uppercased())
    }
}

struct CapitalizedStrategy: LoggerStrategy {
    func log(_ message: String) {
        print(message.capitalized)
    }
}
 

var logger = Logger(strategy: CapitalizedStrategy())
logger.log("my first strategy")  // My First Strategy
logger = Logger(strategy: UppercaseStrategy())
logger.log("my first strategy")  // MY FIRST STRATEGY
logger = Logger(strategy: LowercaseStrategy())
logger.log("my first strategy")  // my first strategy

하지만 strategy pattern을 사용한다면 logger와 log는 직접 건드리지 않고 LoggerStrategy을 준거하는 구조체를 만들어주면 된다.
