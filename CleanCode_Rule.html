TIL - 2021/03/03 CleanCode 형식맞추기

형식을 맞추는 목적

코드작성후 오랜시간이 지나도 정해진 규율이 있다면 구현스타일과 가독성, 유지보수성은 유지된다.

적절한 행 길이를 유지하라

한 파일당 200줄 이하로 구축하는 것이 좋다.

신문기사 처럼 작성하라

이름만 보고도 올바른 모듈을 살펴보고 있는지 알수있고, 소스파일의 첫부분은 고차원 개념과 알고리즘을 설명한다. 아래로 내려갈수록 의도를 세세하게 묘사한다.

개념은 빈 행으로 분리하라

일련의 행 묶음은 하나의 개념을 표현한다. 개념사이에는 빈 행을 넣어 분리해야 마땅하다.

세로 밀집도

서로 밀접한 코드행은 세로로 가까이 놓여야한다.

class ReporterConfig{
    /**
     * Reporter Class Name
     */
    string m_className;

    /**
     * Reporter Property
     */
    List<Property> m_properties = new List<Property>();
    void addProperty(Property property){
        m_properties.add(property);
    }
}

위처럼 의미없는 주석으로 밀접한 코드행을 떨어지게하면 가독성이 떨어진다. 아래와 같이 주석을 없애면, 한눈에 변수1개, 메서드 2개의 클래스라는 것을 알 수 있다.

class ReporterConfig{
    string m_className;
    List<Property> m_properties = new List<Property>();
    void addProperty(Property property){
        m_properties.add(property);
    }
}

수직거리

위에서 말했듯이 서로 밀접한 개념은 세로로 가까이 둬야한다. 하지만 두 개념이 서로 다른 파일에 속한다면 규칙이 통하지않는다. protected변수를 많이 쓰면 소스파일과 클래스를 여기저기 뒤지게 된다.

  • 변수선언 변수는 사용하는 위치에 최대한 가까이 선언한다.
  • 인스턴스변수 인스턴스 변수는 클래스 맨 처음에 선언한다. 잘 설계한 클래스는 대다수의 메서드에서 인스턴스변수를 사용하기 때문이다.
  • 종속함수 한 함수가 다른 함수를 호출한다면 그 두 함수는 세로로 가까이 배치한다. 가능하면 호출하는 함수를 먼저 선언하고, 호출되는 함수를 그 다음 선언한다.

  • 개념적 유사성 친화도가 높을 수록 코드를 가까이 배치한다. 위의 직접적인 종성도 친화도가 높은 것이다. ``` cpp class Assert{ static void assertTrue(string message, bool condition){ if(!condition) fail(message); }

    static void assertTrue(bool condition){ assertTrue(null, condition); }

    static void assertFalse(string message, bool condition){ assertTrue(null, !condition); }

    static void assertTrue(string message, bool condition){ assertFalse(null, condition); }

}

위와 같은 경우 명명법이 같고, 기본기능이 유사하기 때문에 개념적인 친화도가 매우 높다.

### 세로 순서
호출되는 함수를 호출하는 함수보다 나중에 배치하면, 소스 코드 모듈이 고차원에서 저차원으로 자연스럽게 내려간다.

## 가로 형식 맞추기
조사결과 20~60자가 40%의 비율을 차지했지만, 100자나 120자에 달해도 나쁘지 않다.
요즘은 모니터가 크기때문에 한화면에 많이들어간다. 하지만 120자 정도로 행길이를 제한하도록하자.

### 가로 정렬
필자는 한때 아래와 같이 가로정렬을 통해 코드를 작성하였다
``` cpp
class FitNesseExpediter

하지만 이렇게 할 경우 변수유형보다 변수이름에 눈이 가게 되고, 이러한 형식은 보통 코드형식을 자동으로 맞춰주는 툴에서 무시된다. 또한 이렇게 정렬해야될 필요성이 있다는 것은 클래스를 쪼개야된다는 의미이다.

들여쓰기

때로는 간단한 if문, 짧은 while문, 짧은 함수에서 들여쓰기 규칙을 무시하고 싶어지는데, 필자는 항상 제대로 범위를 표현한 코드를 작성한다.

팀 규칙

프로그래머라면 자기가 선호하는 규칙이 있지만, 팀에 속한다면 그 합의된 한가지의 규칙을 따라야한다.

밥아저씨의 형식 규칙

필자의 사용하는 규칙을 표현해주는 코드이다.

public class CodeAnalyzer implements JavaFileAnalysis { 
	private int lineCount;
	private int maxLineWidth;
	private int widestLineNumber;
	private LineWidthHistogram lineWidthHistogram; 
	private int totalChars;
	
	public CodeAnalyzer() {
		lineWidthHistogram = new LineWidthHistogram();
	}
	
	public static List<File> findJavaFiles(File parentDirectory) { 
		List<File> files = new ArrayList<File>(); 
		findJavaFiles(parentDirectory, files);
		return files;
	}
	
	private static void findJavaFiles(File parentDirectory, List<File> files) {
		for (File file : parentDirectory.listFiles()) {
			if (file.getName().endsWith(".java")) 
				files.add(file);
			else if (file.isDirectory()) 
				findJavaFiles(file, files);
		} 
	}
	
	public void analyzeFile(File javaFile) throws Exception { 
		BufferedReader br = new BufferedReader(new FileReader(javaFile)); 
		String line;
		while ((line = br.readLine()) != null)
			measureLine(line); 
	}
	
	private void measureLine(String line) { 
		lineCount++;
		int lineSize = line.length();
		totalChars += lineSize; 
		lineWidthHistogram.addLine(lineSize, lineCount);
		recordWidestLine(lineSize);
	}
	
	private void recordWidestLine(int lineSize) { 
		if (lineSize > maxLineWidth) {
			maxLineWidth = lineSize;
			widestLineNumber = lineCount; 
		}
	}

	public int getLineCount() { 
		return lineCount;
	}

	public int getMaxLineWidth() { 
		return maxLineWidth;
	}

	public int getWidestLineNumber() { 
		return widestLineNumber;
	}

	public LineWidthHistogram getLineWidthHistogram() {
		return lineWidthHistogram;
	}
	
	public double getMeanLineWidth() { 
		return (double)totalChars/lineCount;
	}

	public int getMedianLineWidth() {
		Integer[] sortedWidths = getSortedWidths(); 
		int cumulativeLineCount = 0;
		for (int width : sortedWidths) {
			cumulativeLineCount += lineCountForWidth(width); 
			if (cumulativeLineCount > lineCount/2)
				return width;
		}
		throw new Error("Cannot get here"); 
	}
	
	private int lineCountForWidth(int width) {
		return lineWidthHistogram.getLinesforWidth(width).size();
	}
	
	private Integer[] getSortedWidths() {
		Set<Integer> widths = lineWidthHistogram.getWidths(); 
		Integer[] sortedWidths = (widths.toArray(new Integer[0])); 
		Arrays.sort(sortedWidths);
		return sortedWidths;
	} 
}
