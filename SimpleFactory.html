팩토리 패턴은 생성 패턴으로 객체를 생성할때 사용 되는 디자인패턴이다. 객체를 생성하는 부분을 팩토리 클래스로 따로 빼서 관리하는 것이다.

simple Factory

Simple Factory는 디자인패턴이라고는 할 수 없다고한다. 프로그래밍에서 자주쓰이는 용법..? 그정도라고 보면된다. 그러면 simple Factory는 뭔가?

구조

#include<iostream>
int main()
{
    Pizza objPizza;
    std::string order;
    if(order == "cheesePizza")
        objPizza = new CheesePizza;
    else if(order =="bbqPizza")
        objPizza = new BbqPizza;
    else if(order =="potatoPizza")
        objPizza = new PotatoPizza;
    else
        objPizza = new OriginalPizza;

    Pizza.cook();    
    return 0;
}

Pizza클래스가 있다고 가정하고, Pizza클래스를 상속하는 cheesePizza, bbqPizza, potatoPizza가 있다고 하자. 다형성을 이용하여 만든 클래스를 동적으로 바꿀 수 있도록 코드를 짰다. 하지만 이코드는 피자메뉴가 추가될 경우 계속해서 main()을 바꿔줘야하고 테스트도 다시 해야된다. 여기서 Simple Factory를 사용하면 유지보수성이 좋아지고 캡슐화가 되어 객체지향적인 코딩이 가능하다.

#include<iostream>
class PizzaFactory
{
    Pizza createPizza(std::string order)
    {
        Pizza objPizza;
        if(order == "cheesePizza")
            objPizza = new CheesePizza;
        else if(order =="bbqPizza")
            objPizza = new BbqPizza;
        else if(order =="potatoPizza")
            objPizza = new PotatoPizza;
        else
            objPizza = new OriginalPizza; 
        return objPizza; 
    }
}

int main()
{
    Pizza objPizza;
    std::string order = "cheesePizza";
    PizzaFactory objPizzaFactory;
    objPizzaFactory.createPizza(order);

    Pizza.cook();    
    return 0;
}

위같은 형식이 Simple Factory를 적용한 예이다. 이렇게 코딩을 할 경우 혹시 피자메뉴가 추가되거나하면 PizzaFactory클래스만 변경하면 되기 때문에 main은 건드릴 필요가 없게된다.
