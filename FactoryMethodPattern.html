정의

팩토리 메소드 패턴에서는 객체를 생성하기 위한 인터페이스를 저의하는데, 어떤 클래스의 인스턴스를 만들지는 서브클래스에서 결정하게 만듭니다. 팩토리ㅣ메소드패턴을 이용하면 클래스의 인스턴스를 만드는 일을 서브클래스의 맡기는 것입니다.

Factory Method Pattern은 Simple Factory에서 확장되었다고 생각하면된다. 한 피자가게에 대해서 피자생성부분을 Factory로 만들어서 관리하고 싶다면 Simple Factory로 가능하다.

하지만 한 피자가게가 있고, 그 피자가게의 체인점으로 특색이있는 피자가게들이 문을 열게된다면 Simple Factory에서 확장을 할 필요가 있습니다.

구조

  • Creator : Product를 가지고 원하는 일 을 하기위한 모든 메소드들이 구현되어있다. PizzaStore에 상응
  • ConcreteCreator : 실제로 Product를 생산한다. 즉 각각의자 가게의 피자 인스턴스를 생성한다.
  • Product : Pizza클래스에 해당하며, 이 클래스를 상속하여 여러 종류의 피자를 정의한다.
  • ConcreteProduct : Pizza클래스를 상속하여 BbqPizza, PotatoPizza등 여러 피자들을 정의한다.

장점

  • 객체 생성코드를 한 객체에 집어넣어 중복을 없앰
  • 관리시 Factory객체만 신경쓰면됌
  • 유저입장에서는 인터페이스만 신경써서 사용가능

소스코드

#include<iostream>
#include<string>

// Product
class Pizza
{
public : 
	Pizza() { pizzaName = ""; };
	Pizza(std::string pizzaName) { this->pizzaName = pizzaName; };

	void cook()
	{
		std::cout << pizzaName << " is cooked"<< std::endl;
	}
	void cut()
	{
		std::cout << pizzaName << " is cut" << std::endl;
	}
	void pack()
	{
		std::cout << pizzaName << " is packed" << std::endl;
	}

	std::string pizzaName;
};

// ConcreateProduct
class ChicagoCheesePizza : public Pizza{public:ChicagoCheesePizza() :Pizza("ChicagoCheesePizza") {};};
class ChicagoPotatoPizza : public Pizza{public:ChicagoPotatoPizza() :Pizza("ChicagoPotatoPizza") {};};
class ChicagoBbqPizza : public Pizza{public:ChicagoBbqPizza() :Pizza("ChicagoBbqPizza") {};};
class ThinCheesePizza : public Pizza { public:ThinCheesePizza() :Pizza("ThinCheesePizza") {}; };
class ThinPotatoPizza : public Pizza { public:ThinPotatoPizza() :Pizza("ThinPotatoPizza") {}; };
class ThinBbqPizza : public Pizza { public:ThinBbqPizza() :Pizza("ThinBbqPizza") {}; };


// Creator
class PizzaStore
{
public:
	Pizza* pizza;

	Pizza* orderPizza(std::string pizzaName)
	{
		this->pizza= createPizza(pizzaName);
		
		this->pizza->cook();
		this->pizza->cut();
		this->pizza->pack();

		return this->pizza;
	}

	virtual Pizza* createPizza(std::string ) = 0;
	
};

// ConcreateCreator
class ChicagoPizzaStore : public PizzaStore
{
public:
	Pizza* createPizza(std::string pizzaName) override
	{
		if (!pizzaName.compare("Cheese"))
		{
			return new ChicagoCheesePizza;
		}else if (!pizzaName.compare("Bbq"))
		{
			return new ChicagoBbqPizza;
		}else if (!pizzaName.compare("Potato"))
		{
			return new ChicagoPotatoPizza;
		}
		else
		{
			return new Pizza;
		}
	}
};

class ThinPizzaStore : public PizzaStore
{
public:
	Pizza* createPizza(std::string pizzaName) override
	{
		if (!pizzaName.compare("Cheese"))
		{
			return new ThinCheesePizza;
		}
		else if (!pizzaName.compare("Bbq"))
		{
			return new ThinBbqPizza;
		}
		else if (!pizzaName.compare("Potato"))
		{
			return new ThinPotatoPizza;
		}
		else
		{
			return new Pizza;
		}
	}
};

int main()
{
	PizzaStore* chicagoPizzaStore = new ChicagoPizzaStore;
	Pizza* orderedPizza1;
	orderedPizza1 = chicagoPizzaStore->orderPizza("Cheese");
	
	PizzaStore* thinPizzaStore = new ThinPizzaStore;
	Pizza* orderedPizza2;
	orderedPizza2 = thinPizzaStore->orderPizza("Potato");

	return 0;
}

SimpleFactory와의 차이점은 피자를 만드는 처리가(createPizza)추상 메소드로 정의되어 각각의 자식클래스(ChicagoPizzaStore, ThinPizzaStore)에서 구현하고 있다는 점이다.
