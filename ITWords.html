Daisy-Chain

다중 화면 연결이라고도 하며 여러대의 모니터를 A-B-C의 형식으로 연결해 사용할 수 있는 기능이다. Daisy Chain은 원래 데이지라는 꽃을 연결해서 팔찌나 목걸이를 만드는 것인데, 이러한 형식이 비슷하다는 점에서 이 용어를 쓰는 듯 하다.

IDE

Integrated Device Eletronics 여기서 말하고자하는 IDE는 인터페이스를 의미한다(통합개발환경X) IDE는 EIDE, PATA라고도 불리며, 병렬형식이며 CD-ROM, HDD를 연결하는 인터페이스 규격이다. 전송 속도는 약 최소 3.3MB/s 최대 16.7MB/s 정도로 SATA와는 비교도 안될 정도로 느리다.

SATA

Serial Advanced Technology Attatchment 직렬형식의 인터페이스이며 현재 보통 사용되는 인터페이스가 바로 이것이다. SATA1,2,3가 있고 버전별로 속도가 150,300,600MB/s까지 지원한다고 한다.

SCSI

Small Computer System Interface로 스커지라고 부르며 한국에서는 스카시라고 부른다고한다. 병렬 전송이 특징이며, 고성능과 안정성을 보장하는 인터페이스이다. 예전에는 서버 쪽에서 많이 사용하던 인터페이스이나, 현재는 SAS라는 직렬형태로 넘어간 편이다. 전송속도는 10MB/s정도 이지만, CPU와는 독립적이라 멀티태스킹할 경우 성능이 발휘된다고한다. 메인보드에서 SCSI를 지원하는 경우는 없고, 따로 컨트롤러가 필요하다.

SAS

Serial Attached SCSI 스카시의 직렬버전이고, 보통 서버쪽에서 사용하는 인터페이스이다. SCSI처럼 따로 컨트롤러가 필요하다.

토폴로지

일반적인 의미는 물리적인 배치의 형태로 이루어진 어떤 현장의 종류를 설명하는 것이다.

NAPT (Network Address Port Translation)

NAT(Network Address Translation)의 기능과 같이 OSI 3계층 네트워크 계층에서 사설 IP 주소를 공인 IP 주소로 변환하는 기능을 가진다. 하지만 NAPT의경우 공인IP를 이용하여 다수의 사설IP로 변환가능하다. 이름에서 알 수 있듯이 포트번호를 이용하여 가능케한다. ICMP와같은 패킷들은 port정보를 포함하지않지만 TCP/UDP의경우 port정보를 포함하기때문에 사용이 가능하다.

ICMP(Internet Control Message Protocol)

ICMP는 TCP/IP에서 IP 패킷을 처리할 때 발생되는 문제를 알려주는 프로토콜이다. ping이 이프로토콜을 사용하고있다.

DHCP(Dynamic Host Configuration Protocol)

DHCP란 호스트의 IP주소와 각종 TCP/IP 프로토콜의 기본 설정을 클라이언트에게 자동적으로 제공해주는 프로토콜을 말합니다

PPPoE(Point to Point Protocol Over Ethernet)

PPPoE는 이더넷을 통한 지점 간 프로토콜의 약자입니다. PPPoE는 PPP라고하는 다른 오래된 프로토콜에서 파생 된 네트워킹 프로토콜로, 지점 간 프로토콜을 나타냅니다. 이더넷 프레임의 데이터 부분에 PPP 프레임이 캡슐화되어 전송됨

패킷필터링

네트워크에서는 IP 패킷이라는 단위로 데이터를 송수신하고 있는데, IP 패킷에는 송신하는 데이터 외에 보내는 측과 받는 측의 주소나 통신 절차 등과 같은 정보가 들어 있다. 패킷 필터링에서는 이러한 정보에 대해 조건을 설정하여 통신 가능 여부를 제어한다.

통신계층별 데이터 단위

  • Layer 5 ~ Layer 7 : 메세지,데이터
  • Layer 4 : 세그먼트
  • Layer 3 : 패킷,데이터그램
  • Layer 2 : 프레임
  • Layer 1 : 비트

## FQDN(Fully Qualified Domain Name) FQDN은 명확한 도메인 표기법을 칭한다. 예로 소프트웨어 설치 중 도메인명을 요구하면, YAHOO.COM. 을 입력할지, WWW.YAHOO.COM. 을 입력할지 모호하다. 그래서 이러한 모호성을 피하기 위해 FQDN이란 단어를 사용하며, 이는 Namespace 계층상에서 최종 호스트명을 포함하는 도메인명을 뜻한다. www(호스트명), yahoo.com.(도메인명), www.yahoo.com.(FQDN)

Malware분석방법

악성코드를 분석하는 방법이며 상세,동적,정적,자동화분석이 있다.

  • 상세분석 : 디스어셈블을 통해 어셈블리어로 바꿔 상세분석한다.
  • 동적분석 : 해당파일을 직접 실행하여 나타나는 변화를 모니터링하며 악성코드를 판별하는 방법이다. 가상환경에서 독립된 네트워크를 형성하여 분석도구등을 통해 분석한다.
  • 정적분석 : 파일을 실행하지않고 그 파일자체가 가지고 있는 내용들로 악성코드를 판별하는 방법이다. 파일의 해시나, 사용되는 API,문자열등으로 확인한다.
  • 자동화분석 : 말그대로 자동으로 분석해주는 방법이다. 빠르지만 완벽하지않다.

ARP

ARP(Address Resolution Protocol)의 약자로서 IP주소를 MAC(Media Access Control)주소로 변환해주는 프로토콜이라고 합니다.

공개키 암호화방식

공개키와 개인키를 이용하여 암호화하는 방식. 공개키는 암호화하는데 사용되며, 개인키는 복호화하는데 사용된다. 디지털서명에 이 방식이 사용된다. 개인키가 노출되거나 잃어버릴 경우 암호화된 메세지가 위험에 노출되며, 암호화 및 해독에관련된 복잡한 수학적 계산때문에 대량의 데이터를 처리해야할 경우 매우 느려지게됩니다. SSL(Secure Sockets Layer) 프로토콜도 비대칭 암호화 기능을 사용합니다. 공개 키 암호 방식은 또한 블록체인 및 암호 화폐 기술에서도 두드러지는 특징입니다.

디지털서명

디지털 서명은 디지털 데이터의 진위성과 무결성을 검증하는 데 사용되는 암호 메커니즘입니다. 해싱, 서명, 검증 세 가지 기본 단계로 이뤄지는 경우가 많습니다.

  1. 해싱 : 해싱 알고리즘을 통해 데이터를 제출하여 해시값이 생성(예를 들면 메시지 다이제스트)
  2. 서명 : 메시지 발신자의 개인키를 이용하여 서명
  3. 검증 : 수신자가 공개키를 받아 그 공개키를 이용하여 유효성을 검증

데이터 무결성, 진위성, 부인 방지

DMZ(DeMilitarized Zone)

내부세그먼트와 외부세그먼트로부터 격리된 영역

SIEM(Security Information and Event Management)

네트워크내의 다양한기기나 소프트웨어로부터 로그를 수집해 일괄관리하는 통합로그관리시스템이다. 자동으로 로그 수집,분석을 하며, 사전에 이상을 감지하는것이 가능하다.

C&C서버

봇에 감염된 컴퓨터에 대해서 다른 컴퓨터로의 공격이나 부정한조작을 하도록 외부로부터 지시해 원격조작하는 역할을 맡는 서버이다.

SDN(Software-Defined Networking)

소프트웨어 애플리케이션을 사용하여 네트워크를 지능화하고 중앙에서 제어하거나 ‘프로그래밍’할 수 있는 네트워크 아키텍처 접근법이다. 네트워크상의 다양한 통신기기를 집중적으로 제어하고 네트워크 구성이나 보안설정등을 변경한다.

WAF(Web Application Firewall)

패킷의헤더정보를 검사하는것 뿐만아니라 통신처의 어플리케이션프로그램을 식별해서 통신을 제어한다.

APOP(Authenticated Post Office Protocol)

POP은 메일서버의 메일박스로부터 전자메일을 꺼낼때 사용하는 프로토콜이다. 유저명과 비밀번호에따른 인증을 수행하나, 패스워드를 평문으로 보내는 결점이있다. 이 점을 대처해 해쉬함수등을 사용하여 패스워드송신을 암호화해 안정성을 높인 것이 APOP이다.

POP3S(POP3 over SSL/TLS)

APOP은 패스워드만 암호화하지만, 전송로를 암호화하는 SSL/TLS에의해 POP통신자체를 암호화하는 방식이다.

S/MIME(Secure Multipurposer Internet Mail Extensions)

MIME은 전자메일로 문자뿐만아니라 이미지파일이나, 음성파일등을 다루기위한 규격이다. S/MIME은 MIME의 확장규격으로 전자메일의 내용을 암호화하거나, 디지털서명을 부여하는등 보안대책을 넣은 것이다.

SMTP-AUTH(Simple Mail Transfer Protocol-Authentication)

메일을 송신, 전송하는 프로토콜 SMTP의 확장규격이다. 단순한 메일전송을 수행하는 SMTP에는 인증의 구조가 없기때문에 SMTP-AUTH에서는 송신하는쪽의 메일서버에서 유저명과 패스워드에따른 인증을 수행해, 송신자가 정규의 유저인지 확인이가능하다.

DFD(Data Flow Diagram)

데이터의 흐름에 착목해서 업무처리를 모델화한 다이어그램이다.

  • ○：처리(프로세스), 데이터의가공이나 처리의 내용을 나타낸다.
  • 二：데이터저장(파일), 데이터가 저장되는 파일을 나타낸다.
  • □：외부, 데이터의 발생원이나 데이터의 행선지를 나타낸다.
  • →：데이터흐름, 데이터의 흐름을 나타낸다.

스텁(Stub)

골격만 있는 또는 특별한 목적의 소프트웨어 컴포넌트를 구현한 것. 스텁을 호출하거나 또는 스텁에 의존적인 컴포넌트를 개발하거나 테스트할 때 사용됨. 스텁은 호출된 컴포넌트를 대체 탑다운 방시에서 사용한다.

테스트드라이버(Test Driver)

바텀업방식에서 사용된다. 최하위 모듈로부터 순서대로 모쥴을 결합하는 바텀업테스트에서 적절한 인수를 넘겨 하위 모듈을 호출하는등의 동작을 수행한다.

XP(eXtreme Programming)

아자일개발의 대표적인 방법중하나이며, 유연성이 높은 개발방법이다. 사양의 변경등 변화에 대처해 빠른대응이 가능하다.

특징

  • 코딩과 테스트를 중시
  • 각공정의 피드백을 항상 수행하며, 수정과 재설계하는것을 중시

    종류

  • 테스트기반개발
  • 페어프로그래밍
  • 리팩토링

FP(Function Point)

소프트웨어가 가지는 기능의 개수를 기준으로 그 소프트웨어의 규모를 측정하는 수법이다. 내부논리파일, 외부논리파일, 외부입력, 외부출력, 외부조회기능의 개수에 증분을 곱해 공수를 계산한다.

COCOMO(COnstructive COst MOdel)

공수와 기간등으로 견적을내는 방법중 하나이며, 소스코드의 상정 줄수를 기준으로 프로그래머의 스킬이나 재이용코드량등의 다양한 요인들로부터 개발규모를 파악하고 보정계수를 곱해 공수를 계산하는 방법이다.

MTBF(Mean TIme Between Failures)

평균고장간격, 시스템이 연속해서 정상동작을 하고있는 시간의 평균치

MTTR(Mean Time To Repair)

평균수리시간, 고장으로인해 시스템이 정지하고 수리가 완료되어 재가동할때까지의 시간의 평균치

BCP(Business Continuity Plan)

사업지속계획, 재해등의 리스크가 발생하였을 경우를 대비해, 사업이 계속될 수 있도록 계획이나 복구대책등을 책정하는것이다.

RPO(Recovery Point Objective)

BCP의 지표중 하나로, 사건이 발행 했을때에 과거의 어떤지점까지의 데이터를 복구시킬것인가의 목표치를 의미한다.

RTO(Recovery Time Objective)

BCP의 지표중하나로, 사건이 발생했을때에 언제까지 시스템을 복구시켜야하는가의 목표치이다.
