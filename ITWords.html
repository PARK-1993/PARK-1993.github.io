Daisy-Chain

다중 화면 연결이라고도 하며 여러대의 모니터를 A-B-C의 형식으로 연결해 사용할 수 있는 기능이다. Daisy Chain은 원래 데이지라는 꽃을 연결해서 팔찌나 목걸이를 만드는 것인데, 이러한 형식이 비슷하다는 점에서 이 용어를 쓰는 듯 하다.

IDE

Integrated Device Eletronics 여기서 말하고자하는 IDE는 인터페이스를 의미한다(통합개발환경X) IDE는 EIDE, PATA라고도 불리며, 병렬형식이며 CD-ROM, HDD를 연결하는 인터페이스 규격이다. 전송 속도는 약 최소 3.3MB/s 최대 16.7MB/s 정도로 SATA와는 비교도 안될 정도로 느리다.

SATA

Serial Advanced Technology Attatchment 직렬형식의 인터페이스이며 현재 보통 사용되는 인터페이스가 바로 이것이다. SATA1,2,3가 있고 버전별로 속도가 150,300,600MB/s까지 지원한다고 한다.

SCSI

Small Computer System Interface로 스커지라고 부르며 한국에서는 스카시라고 부른다고한다. 병렬 전송이 특징이며, 고성능과 안정성을 보장하는 인터페이스이다. 예전에는 서버 쪽에서 많이 사용하던 인터페이스이나, 현재는 SAS라는 직렬형태로 넘어간 편이다. 전송속도는 10MB/s정도 이지만, CPU와는 독립적이라 멀티태스킹할 경우 성능이 발휘된다고한다. 메인보드에서 SCSI를 지원하는 경우는 없고, 따로 컨트롤러가 필요하다.

SAS

Serial Attached SCSI 스카시의 직렬버전이고, 보통 서버쪽에서 사용하는 인터페이스이다. SCSI처럼 따로 컨트롤러가 필요하다.

토폴로지

일반적인 의미는 물리적인 배치의 형태로 이루어진 어떤 현장의 종류를 설명하는 것이다.

NAPT (Network Address Port Translation)

NAT(Network Address Translation)의 기능과 같이 OSI 3계층 네트워크 계층에서 사설 IP 주소를 공인 IP 주소로 변환하는 기능을 가진다. 하지만 NAPT의경우 공인IP를 이용하여 다수의 사설IP로 변환가능하다. 이름에서 알 수 있듯이 포트번호를 이용하여 가능케한다. ICMP와같은 패킷들은 port정보를 포함하지않지만 TCP/UDP의경우 port정보를 포함하기때문에 사용이 가능하다.

ARP

ARP(Address Resolution Protocol)의 약자로서 IP주소를 MAC(Media Access Control)주소로 변환해주는 프로토콜이라고 합니다.

ICMP(Internet Control Message Protocol)

ICMP는 TCP/IP에서 IP 패킷을 처리할 때 발생되는 문제를 알려주는 프로토콜이다. ping이 이프로토콜을 사용하고있다.

DHCP(Dynamic Host Configuration Protocol)

DHCP란 호스트의 IP주소와 각종 TCP/IP 프로토콜의 기본 설정을 클라이언트에게 자동적으로 제공해주는 프로토콜을 말합니다

PPPoE(Point to Point Protocol Over Ethernet)

PPPoE는 이더넷을 통한 지점 간 프로토콜의 약자입니다. PPPoE는 PPP라고하는 다른 오래된 프로토콜에서 파생 된 네트워킹 프로토콜로, 지점 간 프로토콜을 나타냅니다. 이더넷 프레임의 데이터 부분에 PPP 프레임이 캡슐화되어 전송됩니다.

패킷필터링

네트워크에서는 IP 패킷이라는 단위로 데이터를 송수신하고 있는데, IP 패킷에는 송신하는 데이터 외에 보내는 측과 받는 측의 주소나 통신 절차 등과 같은 정보가 들어 있다. 패킷 필터링에서는 이러한 정보에 대해 조건을 설정하여 통신 가능 여부를 제어한다.

통신계층별 데이터 단위

  • Layer 5 ~ Layer 7 : 메세지,데이터
  • Layer 4 : 세그먼트
  • Layer 3 : 패킷,데이터그램
  • Layer 2 : 프레임
  • Layer 1 : 비트

## FQDN(Fully Qualified Domain Name) FQDN은 명확한 도메인 표기법을 칭한다. 예로 소프트웨어 설치 중 도메인명을 요구하면, YAHOO.COM. 을 입력할지, WWW.YAHOO.COM. 을 입력할지 모호하다. 그래서 이러한 모호성을 피하기 위해 FQDN이란 단어를 사용하며, 이는 Namespace 계층상에서 최종 호스트명을 포함하는 도메인명을 뜻한다. www(호스트명), yahoo.com.(도메인명), www.yahoo.com.(FQDN)

Malware분석방법

악성코드를 분석하는 방법이며 상세,동적,정적,자동화분석이 있다.

  • 상세분석 : 디스어셈블을 통해 어셈블리어로 바꿔 상세분석한다.
  • 동적분석 : 해당파일을 직접 실행하여 나타나는 변화를 모니터링하며 악성코드를 판별하는 방법이다. 가상환경에서 독립된 네트워크를 형성하여 분석도구등을 통해 분석한다.
  • 정적분석 : 파일을 실행하지않고 그 파일자체가 가지고 있는 내용들로 악성코드를 판별하는 방법이다. 파일의 해시나, 사용되는 API,문자열등으로 확인한다.
  • 자동화분석 : 말그대로 자동으로 분석해주는 방법이다. 빠르지만 완벽하지않다.

공개키 암호화방식

공개키와 개인키를 이용하여 암호화하는 방식. 공개키는 암호화하는데 사용되며, 개인키는 복호화하는데 사용된다. 디지털서명에 이 방식이 사용된다. 개인키가 노출되거나 잃어버릴 경우 암호화된 메세지가 위험에 노출되며, 암호화 및 해독에관련된 복잡한 수학적 계산때문에 대량의 데이터를 처리해야할 경우 매우 느려지게됩니다. SSL(Secure Sockets Layer) 프로토콜도 비대칭 암호화 기능을 사용한다. 공개 키 암호 방식은 또한 블록체인 및 암호 화폐 기술에서도 사용된다.

디지털서명

디지털 서명은 디지털 데이터의 진위성과 무결성을 검증하는 데 사용되는 암호 메커니즘입니다. 해싱, 서명, 검증 세 가지 기본 단계로 이뤄지는 경우가 많습니다.

  1. 해싱 : 해싱 알고리즘을 통해 데이터를 제출하여 해시값이 생성(예를 들면 메시지 다이제스트)
  2. 서명 : 메시지 발신자의 개인키를 이용하여 서명
  3. 검증 : 수신자가 공개키를 받아 그 공개키를 이용하여 유효성을 검증

데이터 무결성, 진위성, 부인 방지

DMZ(DeMilitarized Zone)

내부세그먼트와 외부세그먼트로부터 격리된 영역

SIEM(Security Information and Event Management)

네트워크내의 다양한기기나 소프트웨어로부터 로그를 수집해 일괄관리하는 통합로그관리시스템이다. 자동으로 로그 수집,분석을 하며, 사전에 이상을 감지하는것이 가능하다.

C&C서버

봇에 감염된 컴퓨터에 대해서 다른 컴퓨터로의 공격이나 부정한조작을 하도록 외부로부터 지시해 원격조작하는 역할을 맡는 서버이다.

SDN(Software-Defined Networking)

소프트웨어 애플리케이션을 사용하여 네트워크를 지능화하고 중앙에서 제어하거나 ‘프로그래밍’할 수 있는 네트워크 아키텍처 접근법이다. 네트워크상의 다양한 통신기기를 집중적으로 제어하고 네트워크 구성이나 보안설정등을 변경한다. 종래의 네트워크기기는 제어기능과 데이터전송기능 두가지를 탑재하고있는데, 이를 논리적으로 분리해서 데이터전송을 OpenFlow스위치가 담당하고, 제어기능을가진 OpenFlow컨트롤러가 집중관리한다. 제어기능을 프로그래밍하는 것으로 유연한 네트워크 구축을 수행하는것이 가능하다.

WAF(Web Application Firewall)

패킷의헤더정보를 검사하는것 뿐만아니라 통신처의 어플리케이션프로그램을 식별해서 통신을 제어한다.

APOP(Authenticated Post Office Protocol)

POP은 메일서버의 메일박스로부터 전자메일을 꺼낼때 사용하는 프로토콜이다. 유저명과 비밀번호에따른 인증을 수행하나, 패스워드를 평문으로 보내는 결점이있다. 이 점을 대처해 해쉬함수등을 사용하여 패스워드송신을 암호화해 안정성을 높인 것이 APOP이다.

POP3S(POP3 over SSL/TLS)

APOP은 패스워드만 암호화하지만, 전송로를 암호화하는 SSL/TLS에의해 POP통신자체를 암호화하는 방식이다.

S/MIME(Secure Multipurposer Internet Mail Extensions)

MIME은 전자메일로 문자뿐만아니라 이미지파일이나, 음성파일등을 다루기위한 규격이다. S/MIME은 MIME의 확장규격으로 전자메일의 내용을 암호화하거나, 디지털서명을 부여하는등 보안대책을 넣은 것이다.

SMTP-AUTH(Simple Mail Transfer Protocol-Authentication)

메일을 송신, 전송하는 프로토콜 SMTP의 확장규격이다. 단순한 메일전송을 수행하는 SMTP에는 인증의 구조가 없기때문에 SMTP-AUTH에서는 송신하는쪽의 메일서버에서 유저명과 패스워드에따른 인증을 수행해, 송신자가 정규의 유저인지 확인이가능하다.

DFD(Data Flow Diagram)

데이터의 흐름에 착목해서 업무처리를 모델화한 다이어그램이다.

  • ○：처리(프로세스), 데이터의가공이나 처리의 내용을 나타낸다.
  • 二：데이터저장(파일), 데이터가 저장되는 파일을 나타낸다.
  • □：외부, 데이터의 발생원이나 데이터의 행선지를 나타낸다.
  • →：데이터흐름, 데이터의 흐름을 나타낸다.

스텁(Stub)

골격만 있는 또는 특별한 목적의 소프트웨어 컴포넌트를 구현한 것. 스텁을 호출하거나 또는 스텁에 의존적인 컴포넌트를 개발하거나 테스트할 때 사용됨. 스텁은 호출된 컴포넌트를 대체 탑다운 방시에서 사용한다.

테스트드라이버(Test Driver)

바텀업방식에서 사용된다. 최하위 모듈로부터 순서대로 모쥴을 결합하는 바텀업테스트에서 적절한 인수를 넘겨 하위 모듈을 호출하는등의 동작을 수행한다.

XP(eXtreme Programming)

아자일개발의 대표적인 방법중하나이며, 유연성이 높은 개발방법이다. 사양의 변경등 변화에 대처해 빠른대응이 가능하다.

특징

  • 코딩과 테스트를 중시
  • 각공정의 피드백을 항상 수행하며, 수정과 재설계하는것을 중시

종류

  • 테스트기반개발
  • 페어프로그래밍
  • 리팩토링

FP(Function Point)

소프트웨어가 가지는 기능의 개수를 기준으로 그 소프트웨어의 규모를 측정하는 수법이다. 내부논리파일, 외부논리파일, 외부입력, 외부출력, 외부조회기능의 개수에 증분을 곱해 공수를 계산한다.

COCOMO(COnstructive COst MOdel)

공수와 기간등으로 견적을내는 방법중 하나이며, 소스코드의 상정 줄수를 기준으로 프로그래머의 스킬이나 재이용코드량등의 다양한 요인들로부터 개발규모를 파악하고 보정계수를 곱해 공수를 계산하는 방법이다.

MTBF(Mean TIme Between Failures)

평균고장간격, 시스템이 연속해서 정상동작을 하고있는 시간의 평균치

MTTR(Mean Time To Repair)

평균수리시간, 고장으로인해 시스템이 정지하고 수리가 완료되어 재가동할때까지의 시간의 평균치

BCP(Business Continuity Plan)

사업지속계획, 재해등의 리스크가 발생하였을 경우를 대비해, 사업이 계속될 수 있도록 계획이나 복구대책등을 책정하는것이다.

RPO(Recovery Point Objective)

BCP의 지표중 하나로, 사건이 발행 했을때에 과거의 어떤지점까지의 데이터를 복구시킬것인가의 목표치를 의미한다.

RTO(Recovery Time Objective)

BCP의 지표중하나로, 사건이 발생했을때에 언제까지 시스템을 복구시켜야하는가의 목표치이다.

Context Switching

인터럽트에 의해 다른프로세스로 디스패치될 경우 기존실행중의 프로세스의 상태또는 레지스터값을 교체하는 작업이다.

※ Context란 CPU가 해당프로세스를 실행하기위한 프로세스의 정보들을 이야기한다.

※ Context는 프로세스의 PCB(Process Control BLock)에 저장된다.

※ 저장되는 정보는 프로세스상태, 프로그램카운터, 레지스터(누산기, 스택, 색인레지스터), 프로세스번호가 있다.

EPCglobal네트워크

RFID(Radio Frequency IDentifier)이라는 카드에 들어가있는 IC칩에 정보를 기억시키고 전파에의해 정보교환이나 관리를 행하는 기술을 이용하는 IoT기술중 하나로, 물류 네트워크를 최적화하는데 사용되는 네트워크 아키텍처이다.

CDN(Content Delivery Network)

웹 컨텐츠에 엑세스하는 유저에게 가장 가까운 서버로부터 효율적으로 송신할 수 있도록하는 구조의 네트워크이다.

CAPTCHA(Completely Automated Public Turing test to tell Computers and Humans Apart)

웹페이지의 입력폼등의에서 부정이용을 방지하기위해 사용되는 인증기술이다. 흐릿하게하거나 살짝가린 사진으로부터 인간이 읽을수 있는 문자를 판독시켜 수동으로 입력시킨다. 이것으로 인해 응답자가 인간인것을 판단한다.

하이브리드 암호화시스템

대칭키와 공개키 암호화를 섞은 암호화 시스템이다. 대칭키 암호화시스템은 기밀성을 유지한 통신이 가능한데, 키 배송시 보안상 문제가 있다. 공개키의 경우는 복호화에 사용할 키를 배송할 필요가 없기때문에, 배송상 문제는 없지만, 대칭키암호화방식에 비해 매우 느리고, 중간자 공격(MITM:Man In the Middle attack)에 약하다. ※ 중간자 공격 : 해커가 중간에서 통신을 가로채어 수신자에게는 송신자인 척하고 송신자에게는 수신자인 척 해서 양쪽의 공개키와 실제 암호화에 사용되는 대칭키를 모두 얻어내는 기법.

AES(Advanced Encryption Statndard)

암호화 알고리즘이며 대칭형 암호화에 사용된다. 현재 공모전에서 채택된 레인달(Rijindael)이라는 알고리즘에 해당한다. 레인달알고리즘에서도 암호화 키의 길이가 128, 192, 256비트인 세 가지 종류가 AES 표준으로 지정되었다. 각각 AES-128, AES-192, AES-256으로 불린다.

암호화

  1. 대칭형 암호화 암호화에 사용하는 키가 복호화에 사용하는 키와 같으면 대칭형이다. 통상적으로 비공개키 암호화방식에 사용된다.
  2. 비대칭형 암호화 암호화에 사용하는 키가 복호화에 사용하는 키가 다르면 비대칭형이다. 통상적으로 공개키암호화방식에 사용된다. 암호화는 하면 하나의 키 쌍이 생기고 이 두개의 키는 수학적으로 밀접한 관계를 가지고 있다. 두개의 키를 A,B로 봤을때, A로 암호환 암호문은 B로만 복호화 할 수 있다.

    흐름

  3. 내가 공개키 개인키 한쌍을 생성
  4. 상대방에게 공개키 전송
  5. 상대방은 받은 공개키를 이용해 메세지를 암호화
  6. 상대방은 암호문을 나에게 전송
  7. 나는 암호문을 개인키로 복호화

디피-헬만 키 교환(Diffie–Hellman key exchange)

비밀키를 주고받을 경우 이산대수의 난해함에 안정성을 둔 키교환 방식이다. 엘가말(ElGamal) 방식이 디피-헬만 키 교환을 참고해 만들어졌다. 통신 전체가 악의적인 스니퍼에 의해서 도청되고 있더라도 키 값을 정할 수 있게 해주는 원리는 바로, 키 값을 전달하는 것이 아니라 키 값을 만드는 방법을 전달하기 때문이다.

RSA

현재 SSL/TLS에 가장 많이 사용되는 공개키 암호화 알고리즘이다. 현재 나무위키도 HTTPS에 RSA-2048 인증서를 사용하고 있으며, 전세계 대부분의 인터넷 뱅킹(대한민국 포함)이 이 RSA-2048 암호화를 사용한다. 지금 GPG는 RSA-4096으로 키를 암호화 하지만…암호화할 때에는 마음대로였겠지만 해독할 때에는 아니란다를 기본 모토로 한 공개키 암호체계 중 하나이다. 공개키와 개인키가 한 쌍을 이루며, 공개키로 암호화한 내용은 개인키로만, 개인키로 암호화한 내용은 공개키로만 해독할 수 있다. 엄청 큰 숫자는 소인수분해하기가 힘들다는 것을 이용한다.

메시지 인증 코드(Message Authentication Code, MAC)

해시알고리즘을 사용하여 수정또는 변경사항이 있는지 검출 할 수 있다. 거짓행세는 알 수 없으니 비밀키와 같이 사용해야한다.

타원곡선 암호

타원곡선 암호(楕圓曲線暗號, Elliptic curve cryptography)는 타원곡선 이론에 기반한 공개 키 암호 방식이다. 줄여서 ECC라고 쓰기도 한다. 이산대수의 난해함에 안정성을 둔 키교환방식이며, RSA에 비교하여 암호화는 더 빠르고, 짧은 암호키사이즈로 같은수준의 안정성을 제공한다.

IPsec(Security Architecture for Internet Protocol)

IP패킷의 기밀성과 완전성을 암호기술을 사용해 실현하는 구조이다. OSI기본참조모델의 네트워크층에서 동작하고, 조작을 방지하기위한 패킷인증을 수행하는 인증헤더(AH) 와 인증과 암호화까지 수행하는 암호페이로드(ESP)의 두가지 프로토콜을 포함한다.

SSH(Secure Shell)

네트워크상의 컴퓨터 주변 단말들로부터 접속해 원격조작을 하기위한 통신프로토콜또는 소프트웨어를 말한다. SSH는 Telnet과 다르게, 통신데이터의 도청대책과 통신상대의 위장을 방지하는 구조를 갖추고 있다. SSH에서는 서버에 로그인해서 데이터를 주고받는 통신에앞서(로그인세션) SSH세션이 행해진다.

허니포트(Honeypot)

악성코드나 바이러스, 침입자의 거동을 조사하기위해서 의도적으로 최약성을 가지게하는 시스템또는 네트워크를 말한다.

DMZ(DeMilitarized Zone)

외부네트워크나 내부네트워크로부터 격리된영역을 말한다. 외부에 공개하는 서버를 DMZ에 두면, 외부로부터 부정액세스를 파이어월에의해 차단이 가능하다. 또한 임의로 공개하는 서버가 바이러스에 감염되어버렸을 경우에도 내부 네트워크에까지는 피해가 미치지않는다.

SIEM(Security Information and Event Management, 세큐리티정보이벤트관리)

세큐리티분석소프트의 하나이며, 다양한 네트워크기기나 소프트웨어의 동작상황의 기록을 수집하여 관리를 수행하고, 상관분석의 수법을 이용하여 세큐리티상의 위협을 검출,분석한다.

봇넷(BotNet)

로봇과 네트워크를 결합한 단어로, 악성코드에 감염되어 중앙통제장치로부터 지시를 받는 디바이스로 구성된 네트워크이다. 봇마스터에 의해 원격조종되며, C&C서버 방식을 사용한다.

퍼징(Fuzzing)

검사대상의 제품에 문제를 발생시킬수있는 테스트데이터를 보내 그 반응이나 동작을 감시하는것으로 취약성을 검출하는 수법이다.

Ajax(Asynchronous JavaScript + XML)

브라우저와 웹서버가 JavaScript의 비동기통신을 수행하면서 XML형식의 데이터를 이용하여 동적으로 화면을 다시그리는 구조이다. 화면전체의 전이는 하지않고, 동적인 유저인터페이스를 실현하는것이 가능하다.

CSS(Cascading Style Sheets)

웹페이지의 스타일을 지정하는 언어이며, HTML, XML로 기술되어있는 문서에 CSS를 적용하는것으로 문자의 사이즈나 색, 배경의 그림등의 겉모양을 표현하는것이 가능하다.

RSS(Rich Site Summary

웹사이트의 뉴스나 블로그등의 글의 표제나 요약등을 기술하는 XML베이스의 문서포맷이다. RSS에대응한브라우저나 RSS리더를 사용하는것으로 RSS로 전송하고있는 갱신정보등을 취득해 열람하는것이 가능하다.

기업의 이해관계자

프로젝트매니지먼트에관한 지식을 10개의 대조군으로 분류할 수 있다.

  1. 통합
  2. stakeholder
  3. 스코프 : 프로젝트의 달성목표와 성과물, 활동범위등을 명확하게한다. 그리고 프로젝트의 각페이즈에서 실시하는 작업을 계층적으로 분할하여 파악하고 성과물을 설정한다. 게획프로세스의 작업에있어서 목표 성과물 요구사항 혹은 경계를 포함한 프로젝트의 스코프를 정의하는것이 요구되어진다. 관리프로세스의 작업에있어서 스코프의 변경에의해 발생되는 프로젝트의 기회가되는영향을 최대화하고 위협이되는 영향을 최소화하는것이 요구되어진다.
  4. 자원
  5. 시간
  6. 비용
  7. 리스크 : 프로젝트의 목표에 영향을 주는 리스크를 특정하고, 평가를 진행하여 발생시의 영향도를 최소화하는 리스크대응계획을 책정한다. 또한 리스크에의 대응의 실시상황이나 리스크의 발생징후를 감시해 리스크를 관리한다.
  8. 품질
  9. 조달 : 프로젝트의 활동에 필요한 시스템개발요원, 제품, 서비스를 검토하고 조달계획을세워 프로젝트의 외부로부터 최적화된 공급자를 선정, 취득하고 관리를 수행한다.
  10. 커뮤니케이션 : stakeholder와의 적절한 커뮤니케이션을 수행하는것에의해 프로젝트가 원활하게 진행된다. 구체적으로는 프로젝트의 상황을 분석,평가하여 상황이나 예정을 보고하거나, 프로젝트전체계획의 변경시에 대체안을 제안하고 stakeholder나 상위관리자에게 판단이나 의사결정을 촉진하는등 커뮤니케이션의 필요성을 확실히 만족시키고 커뮤니케이션의 과제가 발생한 경우 그것을 해결한다.

PDM(Precedence Diagramming Method)

액티비티를 노드라고 불리는 사각형으로 표현하고, 그작업순서나 의존관계를 화살표로 나타내는 다이어그램이다. FS(Finish-Start),SS(Start-Start), SF(Start-Finish), FF(Finish-Finish)네가지의 관계를 표현할 수 있다.

SLM(Service Level Management)

제공하는 서비스의 내용과 범위 품질을 명확하게하고 보증해, 문서화한 SLA의 내용이 달성되었는지를 측정하고 수정을 검토하는등 서비스레벨의 유지와 향상에 초점을둔다.

롤포워드(Roll Forward)

트랜잭션 로그에는 데이터 변경 처리가 완벽히 끝난 것으로 기록되어 있으나 해당 내용이 데이터 파일에 미처 반영되지 못했을 때 트랜잭션 로그 내용을 참조해 데이터 변경 작업이 실제 데이터 파일에 반영된다.

롤백(Roll Back)

데이터 변경이 가해지는 쿼리가 실행 되면 트랜잭션 로그에 기록된다. 그런데 해당 트랜잭션이 완전히 끝나지 못하게 되면, 해당 작업은 변경 전 상태로 되돌아간다.

SHA(Secure Hash Algorithm)

NSA에서 만들고 NIST에서 표준으로 채택한 암호화 해시알고리즘이다.
